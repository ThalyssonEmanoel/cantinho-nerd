# Ajustes

## Mapa
- O mapa possui tamanhos diferentes para cada usuário, tal fato interfere nos desenhos com o lápis, figuras geométricas, régua e na criação dos quadrados de marcação, ou seja,
isso interefere na identificação da posição exata dos tokens e a distância de um pont X até o ponto Y, que pode ser diferente em diferentes telas, preciso padronizar isso;

## Grade
- Assim como os mapas, as grades também do mesmo problemas, a adaptação com relação ao tamanho da tela, ela é útil, mas precisa ser adapta juntamente com o mapa;
- Apenas o mestre deve ter a opção de mostrar a grade, quando ele ativa a grade todos da sessão devem conseguir ver ela, ressaltando novamente, ela deve ter o mesmo tamanho 
em todos as telas de tamanhos diferentes.

## Canva de desenho
- Os lápis, figuras e etc, possui um problema, quando a primeira pessoa utiliza o lápis ou a régua, os outros não conseguem ver as marcações, 
a não ser que uma segunda pessoa faça a marcação, com isso apenas as marcações posteriores a da primeira pessoa aparecem, fazendo com que a marcação da primeira pessoa
nunca apareça a não ser que ela desenhe novamente;
- Outro problema, é a exclusão dos desenhos e das medições. Levando em consideração uma situação aonde 3 pessoas desenharam na tela, se uma pessoa apertar na lixeira das opção de configuração, todos os desenhos são apagados e não apenas o dele;

## Névoa de guerra
- Melhor remover essa funcionalidade;

## Sessão
- Caso o usuário atualize a página(F5), sairá da sessão e terá que entrar novamente na mesma;

## Tokens
- Os tokens só são apagados após atualizar a página(F5), no caso, quando clica no token aparece as opção de zoom e emoji, as quais funcionam perfeitamente, mas a lixeira, opção de exclusão, só funciona após fazer o que foi dito no início.;
- O mestre não consegue alterar o nome dos monstros;

### Dados Personalizados
Permitir ao usuário criar dados com faces customizadas (ex: d3, d30, d100) ou dados com faces que contenham texto/símbolos ao invés de números.

### Mesa de Dados Compartilhada
Uma área central onde todos os jogadores podem ver os dados rolando em animação 3D antes de revelar o resultado, tornando o momento mais dramático.

### Histórico Persistente de Rolagens
Manter um registro completo de todas as rolagens de dados de uma sessão, salvo no banco de dados, para consulta posterior. Útil para auditorias e para revisar combates.
